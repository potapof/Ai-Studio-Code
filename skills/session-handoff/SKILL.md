# Session Handoff — переход на новую сессию

## Когда вызывать

Пользователь говорит любую из фраз-триггеров перед завершением сессии.

## Что делает Hermes

1. Проверяет текущее состояние платформы:
   - `docker ps` — какие контейнеры работают
   - `docker exec nocodb-postgres-db pg_isready` — статус БД
   - `hermes cronjob list` — активные cron-задачи
   - `git log --oneline -5` — последние коммиты

2. Проверяет БД:
   - Сколько записей в knowledge_base, skills, standards_library
   - Есть ли незавершённые HANDOFF

3. Формирует файл `~/studio/SESSION-HANDOFF.md`:

```markdown
# Session Handoff — <дата и время>

## Состояние платформы
- Docker: <N контейнеров> работают / <M> остановлены
- PostgreSQL: <доступна/недоступна>
- NocoDB: <URL и статус>
- Portainer: <URL и статус>

## Cron-задачи
- budget-watchdog: <статус>
- knowledge-audit: <статус>

## База знаний
- knowledge_base: <N> записей
- skills: <N> навыков
- standards_library: <N> ADR

## Что сделано в этой сессии
- <конкретный список достижений>
- <закоммичено: N коммитов>

## Что в процессе (не завершено)
- <открытые задачи>
- <блокеры если есть>

## Что делать в следующей сессии
1. <конкретный первый шаг>
2. <конкретный второй шаг>
3. ...

## Последние коммиты
<git log --oneline -5>

## Быстрый старт для нового агента
```
cd ~/studio
git pull
docker compose ps
bash scripts/sync-skills.sh
# Прочитать этот файл: cat SESSION-HANDOFF.md
# Найти прошлые сессии: session_search("студия")
```
```

4. Сохраняет файл и выводит его содержимое пользователю.

## Правило для любого агента в новой сессии

Когда агент (Hermes) запускается в новой сессии и пользователь говорит о «Студии», агент ДОЛЖЕН:

1. Прочитать `~/studio/SESSION-HANDOFF.md`
2. Выполнить `docker compose ps` — проверить контейнеры
3. Выполнить `docker exec nocodb-postgres-db pg_isready` — проверить БД
4. Доложить пользователю: «Платформа: N контейнеров, БД жива. В прошлой сессии сделано X. Продолжаем с Y?»
