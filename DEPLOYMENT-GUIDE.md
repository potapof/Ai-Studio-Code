# Инструкция по развёртыванию «Студии программирования»

> Это пошаговое руководство для пользователя, который не является программистом. Вы сможете развернуть платформу, используя Hermes Agent как вашего помощника. Hermes сделает всю техническую работу — вам нужно только давать команды и отвечать на вопросы.

---

## О чём эта инструкция

В этой инструкции мы рассмотрим, как развернуть платформу «Студия программирования» 2.0 на вашей виртуальной машине Linux Mint. У вас уже установлены VirtualBox, Linux Mint и Hermes Agent. Мы проверим, что ещё нужно, и дадим Hermes команду на развёртывание.

После завершения у вас будет работающая платформа: с базой знаний, приборной панелью (NocoDB), AI-агентом Hermes, который может autonomously выполнять задачи, и интеграцией со Stack Overflow for Agents для получения проверенных технических решений.

## Часть 1. Что у вас уже есть и что нужно проверить

### Что такое «Студия программирования» простыми словами

Представьте, что у вас есть умный помощник, который:

- **Помнит** все задачи, которые вы ему давали, и решения, которые он находил
- **Ищет** ответы на технические вопросы в Stack Overflow (проверенные решения)
- **Пишет** код по вашим описаниям
- **Проверяет** свою работу тестами
- **Открывает** Pull Request на GitHub автоматически
- **Учится** на ошибках — сохраняет опыт для будущих задач

Этот помощник — Hermes Agent. А «Студия программирования» — это инфраструктура вокруг него: база данных для памяти (PostgreSQL), приборная панель для вас (NocoDB), и система автономных циклов (Loop Engineering).

### Что уже установлено

| Компонент | Статус | Что это |
|-----------|--------|---------|
| VirtualBox | ✅ Установлен | Программа для запуска виртуальных машин |
| Linux Mint | ✅ Развёрнута | Операционная система в виртуальной машине |
| Hermes Agent | ✅ Установлен | Ваш AI-помощник |
| Docker | ❓ Нужно проверить | Программа для запуска изолированных сервисов |
| Файлы проекта | ❓ Нужно проверить | Конфигурация платформы |

### Что нужно проверить перед началом

Прежде чем дать Hermes команду на развёртывание, давайте проверим, что всё готово. Откройте терминал в Linux Mint (Ctrl+Alt+T) и выполните команды ниже. Не волнуйтесь — эти команды только проверяют, ничего не меняют.

#### Проверка 1: Какая версия Linux Mint

```bash
cat /etc/os-release | head -3
```

Вы должны увидеть что-то вроде:
```
NAME="Linux Mint"
VERSION="22 (Wilma)"
```

Если версия 22 или новее — отлично. Если старее — нужно обновиться.

#### Проверка 2: Сколько свободного места

```bash
df -h / | tail -1
free -h | head -2
```

Нужно минимум:
- **50 ГБ** свободного места на диске (в колонке `Avail`)
- **8 ГБ** свободной оперативной памяти (в строке `available`)

Если меньше — развёртывание может не получиться.

#### Проверка 3: Работает ли интернет

```bash
ping -c 3 api.deepseek.com
```

Должно показать `3 packets transmitted, 3 received` — значит интернет работает.

#### Проверка 4: Установлен ли Docker

```bash
docker --version
```

- Если показывает `Docker version 24.x` или новее — отлично, Docker установлен.
- Если пишет `command not found` — Docker не установлен. Это нормально, Hermes его установит.

#### Проверка 5: Установлен ли Hermes

```bash
which hermes
hermes --version 2>/dev/null || hermes version 2>/dev/null
```

Должно показать путь к Hermes и его версию. Если не показывает — Hermes не установлен. В этом случае вам нужно сначала установить Hermes (см. документацию на https://hermes-agent.nousresearch.com/docs/).

#### Проверка 6: Есть ли файлы проекта

```bash
ls ~/syncthing-host/studio-docs/ 2>/dev/null && echo "Файлы найдены в syncthing-host"
ls ~/host_shared/studio-docs/ 2>/dev/null && echo "Файлы найдены в host_shared"
ls ~/studio/ 2>/dev/null && echo "Файлы найдены в ~/studio"
```

Файлы проекта — это папка `studio-docs`, которую вы скачали с GitHub или получили другим способом. Если файлы не найдены — их нужно скопировать в виртуальную машину (см. Часть 2).

## Часть 2. Подготовка файлов проекта

### Откуда взять файлы проекта

Файлы проекта — это репозиторий `studio-docs`, который мы создали ранее. В нём содержится:

- `examples/docker-compose.yml` — описание всех сервисов платформы
- `examples/configs/` — конфигурационные файлы для агентов
- `examples/sql/` — скрипты для создания базы данных
- `examples/skills/` — навыки Hermes (включая навык развёртывания)
- `examples/mcp/` — конфиги MCP-серверов
- `scripts/` — bash-скрипты для установки и обслуживания
- `docs/` — документация

### Как скопировать файлы в виртуальную машину

У вас есть несколько способов:

#### Способ 1: Через Syncthing (рекомендуется)

Если вы настроили Syncthing между Windows и Linux Mint:

1. На Windows: поместите папку `studio-docs` в `D:\StudioCode\`
2. Дождитесь синхронизации (2-5 минут)
3. В Linux Mint проверьте: `ls ~/syncthing-host/studio-docs/`

#### Способ 2: Через общую папку VirtualBox

1. На Windows: поместите папку `studio-docs` в `D:\StudioCode\`
2. В Linux Mint: `ls ~/host_shared/studio-docs/`
3. Скопируйте в рабочую директорию: `cp -r ~/host_shared/studio-docs ~/studio`

#### Способ 3: Через git clone (если репозиторий на GitHub)

```bash
cd ~
git clone https://github.com/ВАШ_РЕПОЗИТОРИЙ/studio-docs.git studio
```

#### Способ 4: Через USB-накопитель

1. Скопируйте `studio-docs` на USB-флешку на Windows
2. В VirtualBox: Devices → USB → выберите флешку
3. В Linux Mint: флешка появится в файловом менеджере
4. Скопируйте папку в `~/studio`

### Что должно получиться

После копирования у вас должна быть папка `~/studio` со структурой:

```
~/studio/
├── README.md
├── LICENSE
├── CHANGELOG.md
├── docs/           ← документация
├── examples/       ← конфиги, SQL, навыки
│   ├── docker-compose.yml
│   ├── .env.example
│   ├── configs/
│   ├── mcp/
│   ├── sql/
│   └── skills/
├── scripts/        ← bash-скрипты
└── DEPLOYMENT-GUIDE.md  ← этот файл
```

Проверьте:
```bash
ls ~/studio/
```

## Часть 3. Как дать Hermes проект и команду на развёртывание

### Что такое «навык» (skill) для Hermes

Hermes работает с **навыками** — это файлы в формате Markdown с YAML-заголовком, которые описывают, как выполнить определённую задачу. Навык — это подробная инструкция для Hermes: что делать, в каком порядке, какие команды выполнять, как проверять результаты.

Для развёртывания платформы мы создали специальный навык `deploy-studio`, который находится в `~/studio/examples/skills/deploy-studio/SKILL.md`. Этот навык содержит:

- Пошаговый план развёртывания (9 шагов)
- Все необходимые команды
- Объяснения для пользователя (вас)
- Проверки после каждого шага
- Инструкции что делать при ошибках

### Как загрузить навык в Hermes

Когда вы будете готовы начать развёртывание, выполните в терминале:

```bash
# 1. Перейдите в папку проекта
cd ~/studio

# 2. Зарегистрируйте навык развёртывания в Hermes
hermes skills register examples/skills/deploy-studio/SKILL.md

# 3. Проверьте, что навык зарегистрирован
hermes skills list
```

Вы должны увидеть `deploy-studio` в списке навыков.

### Как дать команду Hermes на развёртывание

Теперь самый важный шаг. У вас есть два варианта:

#### Вариант A: Запуск через CLI (рекомендуется)

В терминале выполните:

```bash
hermes task "Разверни платформу Студия программирования, используя навык deploy-studio. Проверь окружение, установи недостающие компоненты, запусти все сервисы, инициализируй базу данных, подключи MCP-серверы. После завершения покажи отчёт с доступными URL." --skill deploy-studio
```

#### Вариант B: Интерактивный режим

Запустите Hermes в интерактивном режиме:

```bash
hermes
```

И введите команду:

```
Разверни платформу Студия программирования, используя навык deploy-studio.
```

Hermes начнёт работу и будет общаться с вами в чате.

### Что Hermes будет делать (и что вы увидите)

Hermes выполнит 9 шагов. Вот что вы увидите на каждом:

#### Шаг 1: Проверка окружения (5 минут)

Hermes проверит:
- Версию Linux Mint
- Установлен ли Docker
- Свободное место на диске
- Работает ли интернет
- Наличие файлов проекта

Вы увидите сообщения типа:
```
Hermes: Проверяю операционную систему...
Hermes: Linux Mint 22 — отлично!
Hermes: Проверяю Docker...
Hermes: Docker не установлен. Сейчас установлю.
```

#### Шаг 2: Установка Docker (5-10 минут)

Если Docker не установлен, Hermes установит его. Вы увидите:
```
Hermes: Устанавливаю Docker... это займёт 2-3 минуты.
Hermes: [вывод команд установки]
Hermes: Docker установлен! Версия: 24.0.7
```

#### Шаг 3: Подготовка файлов (2 минуты)

Hermes скопирует файлы проекта в рабочую директорию и создаст файл `.env` с настройками. **На этом шаге Hermes попросит вас заполнить несколько полей:**

- `POSTGRES_PASSWORD` — пароль для базы данных (Hermes сгенерирует)
- `DEEPSEEK_API_KEY` — ваш API-ключ от DeepSeek
- `GITHUB_TOKEN` — ваш токен GitHub (опционально)

Hermes объяснит, где получить каждый ключ.

#### Шаг 4: Запуск сервисов (10-15 минут)

Hermes запустит все сервисы через Docker Compose:
```
Hermes: Запускаю сервисы... Docker скачивает образы.
Hermes: [прогресс скачивания]
Hermes: Все сервисы запущены. Проверяю готовность...
Hermes: PostgreSQL готов!
Hermes: NocoDB готов!
```

#### Шаг 5: Инициализация базы данных (3 минуты)

Hermes создаст базу данных `hermes_brain` и установит расширения:
- `pgvector` — для семантического поиска
- `Apache AGE` — для графовых запросов
- Создаст таблицы для знаний, задач, истории

#### Шаг 6: Подключение MCP-серверов (5 минут)

Hermes подключится к:
- `postgres MCP` — прямой доступ к базе знаний
- `Stack Overflow for Agents` — внешний источник знаний (опционально, потребует авторизации в браузере)
- `GitHub MCP` — для создания Pull Request (опционально)

#### Шаг 7: Регистрация навыков (1 минута)

Hermes зарегистрирует все навыки:
- `dependency-update` — обновление пакетов
- `morning-triage` — разбор ошибок
- `flaky-test-fix` — исправление тестов
- `lint-and-fix` — автофиксы стиля
- `pr-drafting` — генерация PR

#### Шаг 8: Финальная проверка (2 минуты)

Hermes проверит, что всё работает:
```
Hermes: Проверяю статус сервисов...
Hermes: ✅ PostgreSQL — OK
Hermes: ✅ NocoDB — OK
Hermes: ✅ Hermes — OK
Hermes: ✅ OpenHands — OK
```

#### Шаг 9: Отчёт

Hermes выведет подробный отчёт с доступными URL и инструкциями что делать дальше.

## Часть 4. Что вам нужно подготовить заранее

Чтобы развёртывание прошло гладко, подготовьте следующее **до начала**:

### Обязательно

1. **API-ключ DeepSeek**
   - Зайдите на https://platform.deepseek.com/
   - Зарегистрируйтесь
   - Создайте API-ключ
   - Скопируйте его (начинается с `sk-`)
   - Положите деньги на баланс (минимум $5)

2. **Пароль для базы данных**
   - Hermes сгенерирует его автоматически, но вы можете придумать свой
   - Минимум 20 символов, с буквами, цифрами и спецсимволами

### Опционально (но рекомендуется)

3. **Токен GitHub**
   - Зайдите на https://github.com/settings/tokens
   - Создайте Personal Access Token (classic)
   - Выберите права: `repo`, `workflow`
   - Скопируйте токен (начинается с `ghp_`)

4. **Аккаунт Stack Overflow**
   - Зарегистрируйтесь на https://stackoverflow.com/
   - Это нужно для авторизации Stack Overflow for Agents

### Не обязательно (для уведомлений)

5. **Slack bot token** — если хотите уведомления в Slack
6. **Sentry DSN** — если хотите отслеживать ошибки в production

## Часть 5. После развёртывания — первые шаги

Когда Hermes закончит развёртывание, вы увидите отчёт с URL-адресами. Вот что делать:

### Шаг 1: Откройте NocoDB (приборную панель)

1. В браузере (в Linux Mint или на Windows через `http://localhost:8080`) откройте: `http://localhost:8080`
2. Создайте admin-аккаунт (email + пароль)
3. Создайте проект "Studio"
4. Подключитесь к базе данных:
   - Database Type: PostgreSQL
   - Host: `postgres-db`
   - Port: `5432`
   - User: `nocodb_user`
   - Password: (из вашего `.env` файла)
   - Database: `hermes_brain`

Теперь вы видите все таблицы базы знаний.

### Шаг 2: Создайте первую задачу

В NocoDB:
1. Найдите таблицу `studio_default.tasks`
2. Создайте новую запись:
   - Title: "Тестовая задача"
   - Description: "Проверка работы платформы"
   - Status: "open"
   - Priority: "normal"

### Шаг 3: Дайте задачу Hermes

В терминале:
```bash
hermes task "Найди задачу 'Тестовая задача' в базе данных и предложи решение" 
```

Hermes:
1. Найдёт задачу через postgres MCP
2. Поищет похожие решения в Stack Overflow
3. Поищет в базе знаний прошлые похожие задачи
4. Предложит решение

### Шаг 4: Настройте Kanban-доску

В NocoDB:
1. Откройте таблицу `studio_default.tasks`
2. Создайте новое представление: Kanban
3. Поле группировки: `status`
4. Колонки: `open`, `in_progress`, `review`, `done`

Теперь у вас есть Kanban-доска для управления задачами.

## Часть 6. Если что-то пошло не так

### Проблема: Hermes не запускается

```bash
# Проверьте статус
docker compose ps hermes

# Посмотрите логи
docker compose logs hermes | tail -30

# Перезапустите
docker compose restart hermes
```

### Проблема: NocoDB не открывается

```bash
# Проверьте, запущен ли NocoDB
docker compose ps nocodb-app

# Проверьте логи
docker compose logs nocodb-app | tail -30

# Если не запущен — запустите
docker compose up -d nocodb-app

# Проверьте доступность
curl http://localhost:8080/api/v1/health
```

### Проблема: PostgreSQL не готов

```bash
# Проверьте логи
docker compose logs postgres-db | tail -30

# Проверьте здоровье
docker exec nocodb-postgres-db pg_isready

# Перезапустите
docker compose restart postgres-db
```

### Проблема: Нет интернета в контейнерах

```bash
# Проверьте egress proxy
docker compose logs egress-proxy | tail -20

# Проверьте DNS
docker exec hermes ping -c 3 api.deepseek.com

# Если не работает — проверьте whitelist
cat ~/studio/examples/configs/squid-whitelist.txt
```

### Проблема: Hermes не может подключиться к MCP

```bash
# Список MCP-подключений
hermes mcp list

# Тест конкретного MCP
hermes mcp test hermes-brain

# Переустановка MCP
hermes mcp remove hermes-brain
hermes mcp add hermes-brain --transport stdio -- \
  npx -y @modelcontextprotocol/server-postgres \
  "postgresql://nocodb_user:ПАРОЛЬ@nocodb-postgres-db:5432/hermes_brain"
```

### Как попросить Hermes о помощи

Если вы застряли, просто спросите Hermes:

```bash
hermes "Я не могу открыть NocoDB. Помоги разобраться."
```

Или:

```bash
hermes "Проверь статус всех сервисов и скажи, что не так."
```

Hermes сам проверит логи, найдёт проблему и предложит решение.

## Часть 7. Полезные команды на каждый день

### Управление сервисами

```bash
# Посмотреть статус всех сервисов
docker compose ps

# Остановить все сервисы
docker compose stop

# Запустить все сервисы
docker compose up -d

# Перезапустить один сервис
docker compose restart hermes

# Посмотреть логи
docker compose logs -f hermes
docker compose logs -f --tail=100
```

### Работа с Hermes

```bash
# Дать задачу
hermes task "Описание задачи"

# Список навыков
hermes skills list

# Список MCP-подключений
hermes mcp list

# Поиск в памяти
hermes memory search "похожие задачи"

# Статус loop
hermes loop list
hermes loop status
```

### Backup и восстановление

```bash
# Создать backup
~/studio/scripts/backup.sh

# Восстановить из backup
~/studio/scripts/restore.sh 2026-07-05
```

### Мониторинг

```bash
# Сколько ресурсов используют сервисы
docker stats

# Размер базы данных
docker exec nocodb-postgres-db psql -U nocodb_user -d hermes_brain -c "
SELECT pg_size_pretty(pg_database_size('hermes_brain'));"

# Количество задач в базе
docker exec nocodb-postgres-db psql -U nocodb_user -d hermes_brain -c "
SELECT COUNT(*) FROM studio_default.tasks;"
```

## Часто задаваемые вопросы

**В: Нужно ли мне программировать?**
О: Нет. Hermes делает всю техническую работу. Вы даёте задачи на русском языке, Hermes их выполняет.

**В: Что если я выключу компьютер?**
О: Сервисы остановятся. При следующем включении VM они запустятся автоматически (если включён `live-restore` в Docker).

**В: Сколько стоит использовать платформу?**
О: Основная статья расходов — API-вызовы к DeepSeek (или другой LLM). Примерно $0.14 за 1M входящих токенов и $0.28 за 1M исходящих. В среднем одна задача стоит $0.05-0.50.

**В: Могу ли я использовать другую LLM вместо DeepSeek?**
О: Да. Поддерживаются OpenAI, Anthropic, и другие. Укажите соответствующий API-ключ в `.env`.

**В: Безопасно ли это?**
О: Да. Все сервисы изолированы в Docker-контейнерах. Внешний доступ ограничен egress-proxy с whitelist доменов. API-ключи хранятся в `.env` файле с правами 600.

**В: Что если я захочу обновить платформу?**
О: Выполните `~/studio/scripts/update-stack.sh`. Скрипт сделает backup, скачает новые образы и перезапустит сервисы.

**В: Могу ли я удалить всё и начать заново?**
О: Да. `docker compose down -v` удалит все контейнеры и данные. После этого можно запустить развёртывание заново.

## Итог

Вы развёртываете платформу, в которой:

1. **Вы** — менеджер, который ставит задачи через NocoDB или CLI
2. **Hermes** — AI-агент, который выполняет задачи, ищет решения, сохраняет опыт
3. **PostgreSQL** — мозг системы, хранит знания и историю
4. **NocoDB** — приборная панель для вас
5. **Stack Overflow for Agents** — внешний источник проверенных решений
